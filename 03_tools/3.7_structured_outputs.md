## 3.7 结构化输出：保证响应格式的可靠性

在生产环境中，一次格式错误就可能引发级联故障——JSON 解析失败、工具调用参数不匹配、多 Agent 通信中断。**结构化输出（Structured Outputs）** 是 Claude API 的一项能力，保证模型响应严格匹配开发者指定的 JSON Schema 或工具定义。

> 💡 结构化输出已正式 GA，支持 Claude Opus 4.7、Opus 4.6、Sonnet 4.6、Sonnet 4.5、Haiku 4.5 及后续模型。

### 3.7.1 为什么需要结构化输出

传统方式下，开发者通过提示词要求 Claude “以 JSON 格式返回”，但模型可能返回多余的 Markdown 包装、遗漏必填字段、或数据类型不匹配。结构化输出通过**约束解码（Constrained Decoding）** 在生成时强制匹配 Schema，从根本上消除格式偏差，且不影响模型推理质量。

### 3.7.2 使用方式

#### JSON Schema Mode

直接在 API 请求中指定 `response_format`：

```python
response = client.messages.create(
    model="claude-sonnet-4-6",
    max_tokens=1024,
    messages=[{"role": "user", "content": "提取以下文本的关键信息..."}],
    response_format={
        "type": "json_schema",
        "json_schema": {
            "name": "extraction_result",
            "schema": {
                "type": "object",
                "properties": {
                    "title": {"type": "string"},
                    "date": {"type": "string", "format": "date"},
                    "entities": {
                        "type": "array",
                        "items": {"type": "string"}
                    }
                },
                "required": ["title", "date", "entities"]
            }
        }
    }
)
```

#### Tool Mode

在工具定义中使用 `strict: true`，保证工具调用参数严格匹配 Schema：

```python
tools = [{
    "name": "create_ticket",
    "description": "创建工单",
    "input_schema": {
        "type": "object",
        "properties": {
            "title": {"type": "string"},
            "priority": {"type": "string", "enum": ["P0", "P1", "P2", "P3"]},
            "assignee": {"type": "string"}
        },
        "required": ["title", "priority"]
    },
    "strict": True
}]
```

### 3.7.3 支持的 Schema 特性

结构化输出支持丰富的 JSON Schema 子集，包括基础类型（string、number、integer、boolean、null）、复合类型（嵌套 object、array）、枚举约束（`enum`）、联合类型（`anyOf`）和必填字段（`required`）。

### 3.7.4 关键应用场景

**数据提取**：从非结构化文本（合同、发票、简历）中提取结构化数据，下游系统无需容错处理。

**多 Agent 通信**：Agent 之间通过结构化消息通信时，格式保证消除了序列化/反序列化失败的风险，显著提升系统稳定性。

**复杂搜索工具**：搜索 Agent 返回的结果严格符合预定义结构，前端渲染无需防御性编程。

### 3.7.5 注意事项

- 结构化输出会略微增加首次 Token 延迟（Schema 编译），后续 Token 生成速度不受影响
- 建议在工具调用场景优先使用 Tool Mode（`strict: true`），在纯数据提取场景使用 JSON Schema Mode
- 对于特别复杂的 Schema（深层嵌套超过 5 层），建议拆分为多次调用
