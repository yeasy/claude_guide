## 8.7 Managed Agents：全托管 Agent 平台

此前，将 Agent 推向生产需要数月的基础设施建设——安全沙盒、状态管理、权限体系、凭证管理、每次模型升级后的 Agent 循环重构。**Managed Agents** 是 Anthropic 于 2026 年 4 月推出的全托管 Agent 执行平台，让开发者将精力聚焦在业务逻辑而非基础设施上。

### 8.7.1 Managed Agents vs. Agent SDK

两者定位不同，互为补充：

| 维度 | Agent SDK（8.6 节） | Managed Agents |
|------|------------------------|----------------|
| **部署位置** | 自有基础设施 | Anthropic 云端 |
| **沙盒与隔离** | 自行实现 | 内置生产级沙盒 |
| **长时间运行** | 需自行处理断线恢复 | 原生支持会话跨断线持久化 |
| **凭证管理** | 自行集成密钥库 | 集中式凭证管理（API Key、OAuth） |
| **治理与审计** | 自行搭建 | 内置权限模型与执行追踪 |
| **多 Agent 协调** | 自行编排 | 内置协调机制（研究预览） |
| **适用场景** | 高度定制、已有基础设施 | 快速上线、降低运维成本 |

> 💡 两者可混合使用：本地用 Agent SDK 快速迭代，上线后切换到 Managed Agents 托管执行。

### 8.7.2 核心能力

#### 生产级沙盒

每个 Agent 会话运行在独立的沙盒环境中，包含进程隔离、网络隔离和存储隔离。开发者无需自行配置容器或虚拟机。

#### 长时间自主运行

传统 API 调用受限于单次请求超时。Managed Agents 支持跨越数天甚至数周的持久化会话——Agent 可以在等待外部事件（如审批、CI 结果）时挂起，事件到达后自动恢复执行。

#### 凭证与权限隔离

通过集中式凭证管理系统，安全地向 Agent 提供 API 密钥和 OAuth 令牌，无需将敏感信息嵌入提示词或代码中。权限模型分三层：

- **Agent 级**：限制 Agent 可访问的工具和资源范围
- **用户级**：终端用户的授权与身份传递
- **审计级**：所有 Agent 操作均可追溯

#### 多 Agent 协调（研究预览）

内置的编排机制支持多个 Agent 并行或串行协作，无需开发者自行实现消息路由和状态同步。

### 8.7.3 性能与案例

内部测试显示，在结构化文件生成任务上，Managed Agents 的成功率比标准提示方式提升最高 10 个百分点，在高难度任务上提升尤为显著。

早期采用者包括：

- **Notion**：在工作空间中部署自定义 Agent，支持并行任务执行
- **Rakuten**：数周内在多个部门上线企业级 Agent
- **Asana**：构建协作式 AI 队友
- **Sentry**：调试 Agent 自动编写补丁并创建 Pull Request

### 8.7.4 何时选择 Managed Agents

以下场景优先考虑 Managed Agents：

- 希望快速从概念验证走向生产（"10 倍提速"）
- 需要长时间运行的自主会话（如多日研究任务）
- 对安全合规有严格要求（内置审计、权限、凭证隔离）
- 团队缺乏 Agent 基础设施运维经验

以下场景仍推荐 Agent SDK：

- 需要完全控制执行环境和模型调用链路
- 已有成熟的容器编排和监控体系
- 需要深度定制 Agent 循环逻辑
