## 8.1 Claude Agent：为什么选择 Claude？

2023 年被称为 “Year of the Chatbot”（聊天机器人元年），而 2025 年则是 “Year of the Agent”（智能体元年）。
为什么有这个转变？因为人们不再满足于让 AI 只会 **说话**，而是要求它能够 **做事**。

> 💡 关于智能体的完整介绍，请参阅《智能体 AI 权威指南》。

### 8.1.1 Agent vs. Chatbot：核心区别

在众多 LLM 中，Claude 被公认为最适合做 Agent 的模型（”The Agentic Model”），关键原因在于三大优势：

#### 1. Tool Use 准确率最高

Claude 在 [Berkeley Function Calling Leaderboard](https://gorilla.cs.berkeley.edu/leaderboard.html) 上长期霸榜。在调用工具时，Claude 极少出现参数填错或幻觉调用的现象。这对 Agent 至关重要——一个错误的 API 调用可能导致整个任务失败。

#### 2. 超长 Context 支持（最高 1M）

Agent 在运行过程中会产生大量的日志、中间状态和执行历史。Claude 的 1M context 能力能够容纳更长的执行轨迹，减少”灾难性遗忘”。这意味着即使是复杂的多步任务，Claude 也能保持对早期信息的准确记忆。

#### 3. Constitutional AI 内置安全机制

[Constitutional AI](../11_safety/11.1_cai.md) 使得 Claude 在面对危险指令（如 “删除所有数据库”）时，拥有内置的道德刹车。这对于拥有执行权的 Agent 来说是 **生死攸关的**——一个失控的 Agent 可能造成真实世界的损害。

### 8.1.2 Agent 的典型流程

一个 Claude Agent 的执行循环如下：

```text
用户目标 → Claude 理解意图 → 规划任务拆解
   ↓
调用 MCP 工具 (如文件访问、API 调用、代码执行)
   ↓
观察工具返回结果
   ↓
根据结果反思是否达成目标 → 决定是否继续或纠正方向
   ↓
重复直至任务完成
```

通过 [MCP (Model Context Protocol)](../04_mcp/README.md)，Claude 可以安全可靠地访问各种工具。

### 8.1.3 Claude Agent 的自动化等级

根据人类干预的程度，Agent 可以分为四个等级：

| 等级 | 名称 | 人类角色 | 典型场景 |
|------|------|--------|--------|
| **L1** | Copilot | 指挥每一步 | “帮我重构这个函数” |
| **L2** | Router | 定义意图，AI 调度工具 | “帮我查天气” → 自动选择 Weather API |
| **L3** | Generalist | AI 自主规划和重试 | “帮我写一个贪吃蛇游戏并运行” |
| **L4** | Autonomous | 无需干预，自主运行 | “每周五自动爬取竞品价格，生成报告” |

本章重点探讨 **L3 和 L4 级别**：AI 具有完整的规划、决策和自我修正能力。

### 8.1.4 构建 Claude Agent 的关键要点

- **Tool Use 可靠性**是 Claude Agent 的基石——工具调用若出错，后续步骤都会失败
- **Long Context 管理**对于多步任务至关重要——随着执行历史增长，如何保持上下文的有效性成为挑战
- **Constitutional AI 安全约束**确保 Agent 不会执行有害操作——这是生产级应用的必要条件
