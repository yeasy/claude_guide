# 2.1 提示词基础：清晰、直接、具体

好的提示词遵循三个基本原则：**清晰（Clear）、直接（Direct）、具体（Specific）**。这听起来简单，但实际上不容易做到。

---

## 原则一：清晰（Clear）

**清晰意味着没有歧义。**

### 反面例子

```
请帮我分析一下这个。
```

问题：
- "这个"是什么？
- "分析"是什么意思？找问题？提建议？总结？

### 正面例子

```
请分析以下 Python 代码，找出潜在的性能问题，并给出优化建议。

代码：
def find_duplicates(lst):
    duplicates = []
    for i in lst:
        if lst.count(i) > 1 and i not in duplicates:
            duplicates.append(i)
    return duplicates
```

---

## 原则二：直接（Direct）

**直接意味着告诉 Claude 该做什么，而不是不该做什么。**

### 反面例子

```
写一篇文章，不要太长，不要太正式，不要用复杂的词汇。
```

问题：双重否定让 Claude 难以确定正确方向。

### 正面例子

```
写一篇 300-500 字的文章，语气轻松友好，使用简单易懂的词汇。
```

---

## 原则三：具体（Specific）

**具体意味着提供足够的细节和约束。**

### 反面例子

```
帮我写一封邮件。
```

问题：
- 写给谁？
- 关于什么？
- 什么语气？
- 多长？

### 正面例子

```
请帮我写一封邮件，具体要求如下：

收件人：我的团队成员（共 5 人）
目的：通知下周一（1月13日）的项目评审会议
内容需包含：
- 会议时间：下周一上午 10:00
- 会议地点：3 号会议室
- 需要准备的材料：项目进度报告、问题清单
语气：专业但友好
长度：100-150 字
```

## 实用模板：任务描述结构

一个完整的任务描述通常包含以下要素：

```
<task>
[简要说明你需要 Claude 做什么]
</task>

<context>
[提供必要的背景信息]
</context>

<requirements>
[列出具体要求和约束]
- 要求 1
- 要求 2
- 要求 3
</requirements>

<output_format>
[指定输出的格式]
</output_format>
```

### 完整示例

```
<task>
将以下英文产品描述翻译成中文
</task>

<context>
这是一款面向中国市场的智能家居产品
目标用户是 25-40 岁的都市白领
</context>

<requirements>
- 保持专业的科技感
- 适当本地化（如单位换算）
- 长度与原文相近
</requirements>

<output_format>
只输出翻译后的中文文本，不需要解释
</output_format>

<source_text>
The SmartHome Hub connects all your devices seamlessly...
</source_text>
```

## 常见错误与改进

### 错误 1：过于简短

```
❌ 写个总结
```

```
✅ 请为以下会议记录撰写一份 200 字左右的执行摘要，
   重点突出决策事项和行动项，使用项目符号格式。
```

### 错误 2：过于模糊

```
❌ 帮我改进这段代码
```

```
✅ 请从以下三个方面改进这段代码：
   1. 可读性：添加注释，改进变量命名
   2. 性能：减少不必要的循环
   3. 健壮性：添加错误处理
```

### 错误 3：隐含假设

```
❌ 用更好的方式重写
```

```
✅ 请使用 async/await 语法重写这段基于回调的代码，
   保持功能不变，并处理可能的错误情况。
```

## 检查清单

在发送提示词之前，检查以下问题：

- [ ] Claude 知道它需要做什么吗？
- [ ] Claude 有足够的上下文信息吗？
- [ ] 成功的标准是否明确？
- [ ] 输出格式是否指定？
- [ ] 有没有使用否定表述？（尝试改为肯定）

---

## 练习

尝试改进以下提示词：

**原始提示词**：
```
帮我写点内容，关于 AI 的
```

**改进后**（参考答案）：
```
请撰写一篇关于 AI 在医疗行业应用的介绍性文章。

目标读者：对技术感兴趣但非专业的普通读者
长度：800-1000 字
结构：
1. 引言：为什么 AI 在医疗领域重要
2. 主要应用场景（3-4 个）
3. 挑战与未来展望
语气：通俗易懂，避免过多专业术语
```

---

## 小结

- **清晰**：消除歧义，让意图显而易见
- **直接**：说该做什么，而非不该做什么
- **具体**：提供足够的细节和约束
